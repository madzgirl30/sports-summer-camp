Security Issue Explanation

This page explains the intentional security issue included for the project requirement.

Intentional Vulnerability

The registration page uses JavaScript to place the camper comment into the page with innerHTML. That can create a reflected cross-site scripting issue if someone types HTML or JavaScript-like content into the form.

How to Explain It

In the video, explain that the vulnerability is only for demonstration. A secure version would use textContent, validate all user input, sanitize comments, and never trust information typed into a form.

Secure Parts

The form uses basic required fields, input types, age limits, and JavaScript validation. The links are local files and should not be broken when the folder structure stays the same.
